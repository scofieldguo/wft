1.接口说明

该接口为抽象类接口，需要合作方APP根据自身需求和产品形态自行并有选择性的实现，该接口的所有方法都由SDK主动发起，APP被动接收。

2.方法目录
方法名称 方法说明
init SDK初始化
getSupportSsid 获取SDK支持的SSID列表
checkWiFiAvailable 监测当前是否有可用WiFi
setDebug 设置debug状态
checkWiFiState 当前WiFi连接状态
connect 连接WiFi功能接口
disconnect 断开连接
release 释放内存
3.方法详情

.init

定义 public void init(Context context,WiFiConnectCallback callback);
功能 SDK初始化
返回类型 无
说明 APP在setDebug方法之后调用本方法，完成内部的初始化工作，并设置APP的回调接口callback。

参数列表

参数名 类型 是否必填 参数说明
context Context Y 调用该方法时的Activity
callback WiFiConnectCallback Y 回调接口句柄

.getSupportSsid

定义 public String[] getSupportSsid();
功能 获取SDK支持的SSID列表
返回类型 字符串数组（String[]）
说明 调用该方法，返回SDK所支持链接的SSID列表，APP只能连接该接口返回的SSID列表中WiFi热点。返回结果是以字符串数组的形式返回，如：{“cmcc”,”chinanet”}。

.checkWiFiAvailable

定义 public int checkWiFiAvailable();
功能 监测当前是否有可用WiFi
返回类型 整型(int)
说明 调用该方法，返回当前可用的WiFi信号强度。如果有多个支持的SSID热点，返回的是信号最好的SSID的信号强度。这个扫描的结果依赖于getSupportSsid()接口所支持的SSID。
【0】无可用WIFI； 【1-5】有可用WIFI，强度从弱到强分1-5,5个等级。

.setDebug

定义 public void setDebug(boolean debug);
功能 设置debug状态
返回类型 -
说明 开启关闭日志，在init之前调用。如果设置为false，需要拥有读写SD卡的权限：
android.permission.WRITE_EXTERNAL_STORAGE
android.permission.READ_EXTERNAL_STORAGE

参数列表

参数名 类型 是否必填 参数说明
debug Boolean Y 设置是否开启日志模式
true：开启,false：关闭

.checkWiFiState

定义 public int checkWiFiState();
功能 当前WiFi连接状态
返回类型 整型(int)
说明 获取当前的WiFi连接状态：【0】未连接 【1】正在连接 【2】已连接

.connect

定义 public void connect(String openId,String openKey,String ssid);
功能 连接WiFi功能接口
返回类型 -
说明 openId与openKey是为鉴权而设置的参数；如果不需要鉴权，需要保证openId的唯一性，openKey设置为NULL。连接过程会通过onConnectStateChange()回调函数通知。
SSID值可以根据APP的产品形式而定：
1）如果APP只是需要连接WiFi，并不要求连接指定WiFi热点（如产品形式为一键连接），这时SSID可以指定为NULL，SDK根据自身逻辑自动选择热点连接。
2）如果APP自行扫描并根据getSupportSsid()返回的结果进行过滤（如产品形式为一个WiFi列表，有用户自行选择连接某个WiFi热点），这时就要把指定的SSID作为参数传入，此时SSID不能为NULL。

参数列表

参数名 类型 是否必填 参数说明
openId String Y 要保证其唯一
openKey String N 不需要鉴权，设为NULL
ssid String N 要连接的WiFi的名称，如果不指定，SDK根据自身逻辑自动选择热点连接

.disconnect

定义 public void disconnect();
功能 断开连接
返回类型 -
说明 当不在使用WiFi时调用该方法断开当前连接的运营商热点。通过onDisconnectResult()回调函数降结果通知给APP。

.release

定义 public void release();
功能 释放内存
返回类型 -
说明 该方法对应init方法，当APP结束对SDK的使用，调用本方法释放内存，置空callback。需要注意的是，调用本方法时，如果WiFi为连接状态，则断开连接，如果为正在连接状态，则会取消继续连接。所以，只有在确定不再使用SDK时，在去调用这个方法。
